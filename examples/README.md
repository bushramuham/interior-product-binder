# Example Schedules

Each `.xlsx` file here is a small schedule export in the real-world format
(`KEY | DESCRIPTION | PG | PATH | DESCRIPTION2 | DESCRIPTION3`) that exercises a
specific part of the tool. The sample spec-sheet PDFs they reference live in
`pdfs/`. All files are generated by `scripts/make_examples.py`.

Run each one from the repo root:

| File | Demonstrates | Command |
| --- | --- | --- |
| `example_basic.xlsx` | Happy path: two products, one local PDF each | `python main.py examples/example_basic.xlsx` |
| `example_grouping.xlsx` | Multiple KEYS (U-36/U-37/U-03) sharing one spec sheet → one combined binder page | `python main.py examples/example_grouping.xlsx` |
| `example_mixed_sources.xlsx` | Mix of local PDFs and real public webpage URLs | `python main.py examples/example_mixed_sources.xlsx` |
| `example_errors.xlsx` | A missing local file and an unreachable URL → per-product error blocks, run still completes | `python main.py examples/example_errors.xlsx` |

To regenerate everything in this folder:

```bash
python scripts/make_examples.py
```
